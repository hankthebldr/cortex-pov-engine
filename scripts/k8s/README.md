# scripts/k8s

Generated Kubernetes YAML manifests land here. One manifest per scenario.

Each manifest is generated by push_generator.py and extends the cdr_base.yml template
pattern from sources/CDR/cdr.yml. Simulation begins automatically on `kubectl apply`.
Containers in the manifest correspond to scenario phases — each step maps to a container
init sequence or Job step.

Files are named: `{scenario_id}-{timestamp}.yml`
